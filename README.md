## 📖 About The Project

This project was developed as part of the **ITM Skills University** curriculum (Case Study: *85. MediTravel*). The goal was to design a comprehensive 5-page medical tourism portal from scratch using core web technologies, focusing on semantic structure, responsive layouts, and interactive DOM manipulation without the use of external frameworks.

## ✨ Key Features

* **Dynamic Cost Comparison:** Compare treatment prices, hospital ratings, and recovery times side-by-side across 6 different countries using JavaScript object rendering.
* **Interactive Doctor Profiles:** Clickable doctor cards that trigger a custom-built modal popup displaying detailed specialist information.
* **Real-Time Treatment Filtering:** Sort through various medical procedures based on destination country and medical specialty instantly.
* **Form Validation:** Client-side JavaScript validation for consultation requests ensuring required fields and proper email formats are submitted.
* **Fully Responsive UI:** Built with CSS Flexbox and Grid to adapt seamlessly from large desktop monitors down to mobile devices. Includes a custom JavaScript mobile hamburger menu.
* **Modern UI/UX Enhancements:** Smooth scrolling, `@keyframes` fade-in animations, interactive hover transitions, and input focus highlights.

---

## 🛠️ Tech Stack

* **HTML5:** Semantic structuring (`<header>`, `<nav>`, `<section>`, `<article>`) and accessible form elements.
* **CSS3:** Custom layout systems (Flexbox & CSS Grid), media queries, and animations.
* **JavaScript (Vanilla/ES6+):** DOM manipulation (`querySelector`, `getElementById`), Event Listeners, and dynamic rendering.

---

## 📂 Project Structure

The portal consists of 5 interconnected pages:
1. `index.html` - Hero section, value propositions, and featured treatments.
2. `treatments.html` - Dynamic treatment cards with country/specialty filters.
3. `compare.html` - Interactive side-by-side medical cost comparison tool.
4. `doctors.html` - Global specialist directory with interactive modal popups.
5. `contact.html` - Consultation request form with active validation.
